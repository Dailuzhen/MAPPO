| 指标 | V5 (ep850) | V7 (ep1300) | V8 (ep1350) |
|------|-----------|-------------|-------------|
| 训练评估成功率 | 80.0% (20场景) | 85.0% (20场景) | 77.5% (40场景) |
| **泛化验证成功率** | **33.3%** (135场景) | **73.5%** (200场景) | **73.8%** (160场景) |

| 总训练时间 | ~3h | ~8h | ~4h |

## 二、V5 模型基线

### 2.1 训练配置

- 训练模式：仅 Phase B（PPO），Phase A 来自 full_training_v13
- Phase B：1000 episodes，64 并行环境
- 网络：MLP 256 x 4 层，obs_radius=2（5x5 视野，31维观测）
- 评估：20个 Phase A 历史场景（含A*泄漏）

### 2.2 训练结果

- 训练评估最高：80.0%（ep750, ep850）
- 训练评估平均：57.6%

### 2.3 泛化验证（ep850，135场景，A*完全禁用）

| 地图尺寸 | 成功率 | 碰撞次数 |
|---------|--------|---------|
| 8x8 | 46.7% (7/15) | 148 |
| 10x10 | 53.3% (8/15) | 0 |
| 12x12 | 33.3% (5/15) | 142 |
| 16x16 | 40.0% (6/15) | 2 |
| 20x20 | 40.0% (6/15) | 0 |
| 22x22 | 6.7% (1/15) | 1 |
| **综合** | **33.3% (45/135)** | **295** |

### 2.4 核心问题

1. 训练评估与泛化验证差距达 47 个百分点（80% vs 33%）
2. A*泄漏：评估时使用 Phase A 历史场景，环境可能仍有 A* 辅助
3. 碰撞严重：8x8 碰撞 148 次，12x12 碰撞 142 次
4. max_steps=100 对大地图不足
5. 22x22 地图成功率仅 6.7%

## 三、V5 → V7 改进

### 3.1 改进内容

| 改进项 | V5 | V7 | 说明 |
|-------|----|----|------|
| A*泄漏修复 | 未修复 | 已修复 | 评估时完全禁用A*动作覆盖 |
| 评估方式 | Phase A历史场景 | 纯随机场景 | 消除评估偏差 |
| 并行环境数 | 64 | 128 | 更多训练数据 |
| Phase A模型选择 | 直接使用最终模型 | 选择最佳模型(ep180, 69%) | 避免使用退化的最终模型 |
| 训练episodes | Phase B 1000ep | Phase A 200ep + Phase B 2000ep | 完整两阶段训练 |
| 课程学习 | 有 | 4阶段(8-12→8-16→8-20→8-22) | 结构化课程 |
| 卡住检测 | 无 | looped_to_start 修复 | 准确区分成功vs卡住 |
| 断点续训 | 无 | 支持 | 训练中断可恢复 |

### 3.2 效果提升

| 验证指标 | V5 | V7 | 提升幅度 |
|---------|----|----|---------|
| 综合成功率 | 33.3% | 73.5% | **+40.2pp** |
| 8x8 | 46.7% | 84.0% | +37.3pp |
| 12x12 | 33.3% | 76.0% | +42.7pp |
| 16x16 | 40.0% | 66.0% | +26.0pp |
| 20x20 | 40.0% | 68.0% | +28.0pp |

### 3.3 V7 残留问题

1. **分布偏移**：阶段4(8-22)训练导致小地图性能从84%降到78%，大地图占比过高
2. **评估方差大**：20场景评估噪声高（ep400评估90% vs 验证实际66%）
3. **超时是主要失败模式**：53次失败中53次是超时，智能体在大地图上"迷路"
4. **步数效率偏低**：成功案例步数比约1.3-1.4x（比最优路径多30-40%）
5. **未追踪全局最优模型**：最终模型(ep2000, 65.5%)远不如ep1300(73.5%)

## 四、V7 → V8 改进

### 4.1 改进内容

| 改进项 | V7 | V8 | 目的 |
|-------|----|----|------|
| obs_radius | 2 (5x5=25维) | 3 (7x7=49维) | 增大视野，减少大地图迷路 |
| 奖励-距离惩罚 | -0.05 * manhattan | -0.1 * manhattan / (2*grid_size) | 归一化，消除地图尺寸影响 |
| 奖励-步数惩罚 | 无 | -0.01/步 | 鼓励高效到达 |
| 奖励-抗循环 | 检查最近5步，-0.1 | 检查最近20步，-0.1*重复次数 | 更强的反循环信号 |
| A*进度奖励 | +0.1/-0.02 | +0.2/-0.05 | 更强的路径引导信号 |
| 课程采样 | 均匀随机[min,max] | 分层桶采样（等概率选桶） | 消除大地图分布偏移 |
| 评估场景数 | 20个随机 | 40个随机+按尺寸分组统计 | 降低评估方差 |
| 最优模型追踪 | 无 | 自动保存best_model/ | 不错过训练过程最优模型 |
| 并行环境 | 128 | 256 | 加速训练 |
| ppo_epoch | 10 | 5 | 配合2x数据量，加速训练 |
| entropy_coef | 0.03 | 0.05 | 鼓励更多探索 |
| 日志输出 | 冗余(MAP CHANGE等) | 精简+nohup不中断 | 关键信息不被淹没 |

### 4.4 V7 vs V8 对比

| 验证指标 | V7 (ep1300) | V8 (ep1350) | 变化 |
|---------|-------------|-------------|------|
| **综合成功率** | **73.5%** | **73.8%** | +0.3pp |
| 8x8 | 84.0% | 95.0% | **+11.0pp** |
| 10x10 | - | 90.0% | 新增 |

| 训练时间 | ~8h | ~4h | **缩短50%** |

### 4.5 V8 关键发现

**改善点**：
1. **步数效率显著提升**：V7 步数比 1.3-1.4x → V8 约 1.0x，几乎逼近A*最优路径
2. **小地图能力大幅增强**：8x8 从 84% 提升到 95%，10x10 达到 90%
3. **大地图(20x20)突破**：从 68% 提升到 90%（ep1350）
4. **训练速度翻倍**：256并行 + ppo_epoch=5，总时间从8h降到4h
5. **最优模型自动追踪**：不再遗漏最佳checkpoint

**未解决的问题**：
1. **中等地图波动大**：12x12 在不同checkpoint间从 55% 到 80%，不稳定
2. **22x22 地图瓶颈**：最高仅 70%（ep700），多数在 50-55%
3. **阶段4仍有退化**：综合从阶段3的71.5%降到阶段4的62.2%
4. **综合成功率未突破**：73.8% vs 73.5%，V8整体没有超越V7

### 各版本核心贡献

| 版本 | 核心贡献 | 解决的关键问题 |
|------|---------|-------------|
| V5→V7 | 修复A*泄漏、纯随机评估、课程学习 | 评估虚高、泛化差 |
| V7→V8 | 增大视野、奖励归一化、均衡课程、加速训练 | 大地图迷路、步数低效、训练慢 |

### 未来改进方向

1. **22x22 地图专项优化**：可能需要更大 obs_radius 或 RNN 记忆机制
2. **中等地图稳定性**：12x12 波动大，可能需要更多该尺寸的训练样本
3. **阶段4退化**：考虑更细粒度的课程阶段或训练后期降低学习率
4. **综合突破85%**：需要更强的反循环机制或网络结构改进（如注意力机制）
